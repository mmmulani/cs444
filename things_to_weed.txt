As per http://java.sun.com/docs/books/jls/first_edition/html/8.doc.html#41652,
the name of a constructor must match the name of the class that contains it.

Cast expressions followed by a unary expression should be weeded out. We extend
the cast expression to allow general expressions to make parsing easier.
(As per http://java.sun.com/docs/books/jls/first_edition/html/19.doc.html)
e.g. (1 + 1) a

Check modifiers based on context, e.g. no native classes

Multidimensional arrays in primary expressions.
e.g. a[1][2]

Catch any oct, hex or long literals.
